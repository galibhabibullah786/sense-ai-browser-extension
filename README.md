# SenseAI Browser Extension

Chrome/Chromium browser extension for **SenseAI** — provides real-time website trust analysis, dark-pattern blocking, and AI-powered security explanations directly in your browser toolbar.

## Features

- **Real-time Trust Analysis** — Analyse any website's security and privacy signals instantly.
- **Trust Score Gauge** — Visual 0–100 gauge with color-coded verdict (`safe`, `warning`, `danger`).
- **Signal Breakdown** — Detailed breakdown of SSL/TLS, headers, cookies, trackers, and fingerprinting.
- **AI Explanations** — Natural-language explanations generated by the backend LLM.
- **Dark Pattern Blocking** — Automatically blocks navigation to system-blocklisted domains. Blocklist is cached locally for **offline blocking** and refreshed every 6 hours.
- **Backend-First Analysis** — Trust scores are always generated by backend rules (no local trust-score simulation).
- **Account Linking (Optional Sync)** — Link to a SenseAI dashboard account via one-time code to sync sessions to the web app.
- **Bidirectional Unlink Sync** — Unlinking from the extension revokes the backend token; unlinking from the dashboard disconnects the extension in real time via WebSocket.
- **Token Refresh** — Access tokens are automatically refreshed using stored refresh tokens, surviving browser restarts.
- **Offline Caching** — Caches analysis results in `chrome.storage.local` for instant re-display.
- **Resilient Content Script Injection** — Automatically retries with supported JS injection and falls back to safe minimal signals when injection fails.
- **Dark / Light Mode** — System-aware theme with manual toggle.

## Getting Started

### Prerequisites

- **Node.js** ≥ 18
- **pnpm** (recommended)

### Installation

```bash
cd sense-ai-extension
pnpm install
```

### Development Build

```bash
pnpm dev        # Build with source maps + HMR via CRXJS
```

### Production Build

```bash
pnpm build      # Optimised production build → dist/
```

## Loading in Chrome

1. Navigate to `chrome://extensions/`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked**
4. Select the `dist/` folder from this project
5. Pin the SenseAI extension to your toolbar (puzzle-piece icon → pin)

> Also works in Edge, Brave, Arc, and other Chromium-based browsers.

## Backend Integration

The extension connects to the SenseAI backend via **Socket.io** WebSocket for real-time analysis.

### Linking your account

1. Log in to the SenseAI dashboard
2. Navigate to **Link Extension** (`/link-extension`)
3. Copy the generated link code (e.g. `TRUST-A1B2C3`)
4. Open the extension popup → **Link Account** → paste the code

Once linked, the extension will:
- Send collected signals to the backend in real time
- Receive trust scores and AI explanations via WebSocket
- Fetch and cache the system blocklist for dark-pattern blocking
- Sync analysis sessions to your dashboard

### Without linking

The extension still performs backend analysis when not linked.  
In this mode:
- Results are returned to the extension popup
- Sessions are **not** synced to dashboard/frontend websockets
- Footer status shows backend connectivity and link status separately

### If backend is unavailable

- Trust analysis does **not** use fake simulated scores
- The popup shows a user-friendly error message
- AI explanation can still use template-based fallback when explanation text is missing

## Browsing Protection

SenseAI now includes active browsing protection that blocks dangerous websites:

### Blocklist-Based Blocking
- Known dark-pattern domains from the system blocklist are blocked on navigation
- The extension caches the blocklist locally for offline protection
- User personal blocklist entries (managed from the dashboard) are also enforced

### Trust-Based Blocking
- After clicking "Analyze This Site", if the trust score is below 25 or the verdict is "danger", the page is automatically blocked
- Users see a dedicated blocking page with:
  - The blocked URL and reason
  - The trust score and verdict badge
  - A "Go Back to Safety" button
  - A "Proceed Anyway" option (with 5-second countdown) for non-blocklist blocks

### Testing Browsing Protection

To test the blocking feature from the customer end:

1. **Test blocklist blocking:**
   - Add a domain to your personal blocklist via Dashboard → Settings → Blocked Domains
   - Navigate to that domain — you should see the block page immediately

2. **Test trust-based blocking:**
   - Navigate to a website with known poor security practices (e.g., an HTTP-only site without security headers)
   - Click "Analyze This Site" in the extension popup
   - If the trust score is below 25, the page will be redirected to the block page

3. **Test bypass:**
   - On the block page for trust-based blocks, click "Proceed Anyway"
   - Wait for the 5-second countdown
   - You'll be redirected back to the original site

4. **Test offline blocking:**
   - Disconnect from the internet
   - Navigate to a domain in the cached blocklist
   - The block page should still appear (using local cache)

## Theme Support

The extension popup supports light/dark mode:
- **Auto**: Follows system preference
- **Manual**: Toggle via the sun/moon icon in the popup header
- The blocked page also respects dark mode preferences

The blocked page uses CSS custom properties for theming, making it easy to customize colors.

## Permissions

| Permission | Reason |
| --- | --- |
| `activeTab` | Access the current tab for analysis |

| `cookies` | Read cookie information for security analysis |
| `webRequest` | Detect trackers, redirects, and TLS information |
| `scripting` | Inject content scripts for signal collection |
| `tabs` | Read tab URLs for domain identification |
